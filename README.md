**Nearest** — наиболее быстрый, так как использует простое округление координат

**Bilinear** — расчет по 4 пикселям и взвешенное суммирование

**Bicubic**  — самый долгий, расчет по 4 пикселям и кубическая функция 

| Метод | Среднее время |
| :--- | :---: |
| Nearest | 55 мс |
| Bilinear | 135 мс |
| Bicubic | 850 мс |

Если угол 0°, то PSNR = 1e10 дБ. Так как при повороте на 0° изображение не изменяется, следовательно полная идентичность и бесконечный PSNR.

Наименьшие показатели PSNR при диагональных поворотах.

При угле 360° = 0° так как есть преобразование, следовательно сохраняется ошибка интерполяции и PSNR конечен.

| Метод | Диапазон PSNR |
| :--- | :---: |
| Nearest | 21.6 – 31.1 дБ |
| Bilinear | 27.3 – 29.8 дБ |
| Bicubic | 27.9 – 33.4 дБ |

**Сравнение по времени и качеству:**
Nearest  < Bilinear < Bicubic
